## Description

Implement admin CRUD endpoints for form templates. This is the core backend for the template system — it enables the admin UI (Task 5) to create, read, update, and delete form templates. All routes are admin-only (`role === 'ADMIN'`). This task also creates the `buildZodSchema(fields)` utility in `src/lib/form-validation.ts`, which is later consumed by the public submit endpoint (Task 3).

The three files to create:

1. `src/app/api/templates/route.ts` — `GET` (list all templates) and `POST` (create template). POST auto-inserts a protected "Company Name" field (first field of the first step) and validates the full body with Zod.
2. `src/app/api/templates/[id]/route.ts` — `GET` (single template with steps/fields), `PATCH` (full-replacement update), `DELETE` (cascade via Prisma). PATCH is full-replacement: delete all existing steps/fields and insert new ones in a single transaction. If the incoming PATCH body omits the `isProtected` Company Name field, return 400.
3. `src/lib/form-validation.ts` — exports `buildZodSchema(fields: FormField[])` utility that constructs a Zod object schema from an array of `FormField` records. Used by Task 3's submit endpoint.

## What to do

1. Create `src/lib/form-validation.ts`:
   - Export `buildZodSchema(fields: FormField[])` that returns a `z.ZodObject`.
   - Map each `FieldType` to a Zod schema: `TEXT`/`TEXTAREA` → `z.string().max(10000)`, `EMAIL` → `z.string().email()`, `URL` → `z.string().url()`, `NUMBER` → `z.number().finite()`, `CHECKBOX` → `z.boolean()`, `FILE` → `z.string()` (value is stored as JSON string by submit endpoint).
   - Apply `.optional()` for fields where `required === false`.
   - Key each field by `fieldKey`.

2. Create `src/app/api/templates/route.ts`:
   - `GET`: verify session + `role === 'ADMIN'` (401/403). Return all templates ordered by `createdAt DESC` — include step count and field count but not full nested structure (keep response lean for list view).
   - `POST`: verify session + role. Parse and validate body with Zod: `name: z.string().min(1).max(255)`, `steps` array with `title: z.string().min(1).max(255)` and `fields` array (label max 255, placeholder max 500, fieldKey auto-generated from label). After validation, auto-prepend Company Name protected field to the first step's fields list before inserting. Use `prisma.formTemplate.create` with nested `steps > fields` create. Return 201 with created template.

3. Create `src/app/api/templates/[id]/route.ts`:
   - `GET`: verify session + role. Return template with full `steps > fields` nested structure ordered by `order`.
   - `PATCH`: verify session + role. Validate body same schema as POST. Check that the incoming `steps[].fields[]` array still contains all `isProtected` fields from the existing template — if any protected field is absent, return 400 `"Company Name field cannot be removed"`. Execute full-replacement in a `prisma.$transaction`: delete all `FormStep` records for this template (cascades to `FormField` via Prisma cascade), then create new steps + fields from the request body. Preserve `isProtected: true` on the Company Name field. Return updated template.
   - `DELETE`: verify session + role. Delete template by id (Prisma cascade handles steps/fields). `FormSubmission.templateId` becomes null via `SetNull`. Return 204.

## Details

**Files to create:**

- `src/app/api/templates/route.ts` — GET list + POST create; auth pattern from `src/app/api/merchants/route.ts`
- `src/app/api/templates/[id]/route.ts` — GET single + PATCH full-replace + DELETE; same auth pattern
- `src/lib/form-validation.ts` — `buildZodSchema(fields)` utility; no auth, pure function

**Auth pattern (from `src/app/api/merchants/route.ts`):**
```ts
const session = await getServerSession(authOptions);
if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
if (session.user.role !== "ADMIN") return NextResponse.json({ error: "Forbidden" }, { status: 403 });
```

**Prisma import:** `import { prisma } from "@/lib/db"` — singleton, no instantiation needed.

**Zod limits (Decision 10):**
- `name` / `label` → `z.string().min(1).max(255)`
- `placeholder` → `z.string().max(500).optional()`
- Text/textarea response values → `z.string().max(10000)`

**Company Name auto-insert (Decision 3 + POST flow):**
After Zod validation passes, before `prisma.formTemplate.create`, prepend to `steps[0].fields`:
```ts
{ label: "Company Name", fieldKey: "company-name", type: "TEXT", required: true, isProtected: true, order: 0 }
```
Then renumber `order` of the other fields starting from 1.

**`fieldKey` generation:** auto-generate from `label` using kebab-case: `label.toLowerCase().replace(/[^a-z0-9]+/g, '-').replace(/(^-|-$)/g, '')`. If caller provides `fieldKey`, use as-is.

**PATCH full-replacement (Decision — PATCH template strategy):**
Use the **interactive transaction form** (not the array form — array form does not guarantee execution order):
```ts
await prisma.$transaction(async (tx) => {
  await tx.formStep.deleteMany({ where: { templateId: id } });
  // FormField cascade-deletes via Prisma onDelete: Cascade
  // FieldResponse.fieldId becomes null via SetNull — existing submissions unaffected
  for (const [stepIdx, step] of steps.entries()) {
    const createdStep = await tx.formStep.create({ data: { templateId: id, title: step.title, order: stepIdx } });
    for (const [fieldIdx, field] of step.fields.entries()) {
      await tx.formField.create({ data: { ...field, stepId: createdStep.id, order: fieldIdx } });
    }
  }
});
```

**Protected field removal check (Decision 9 / PATCH guard):**
Before executing the transaction, query existing protected fields: `prisma.formField.findMany({ where: { step: { templateId: id }, isProtected: true } })`. Verify each protected `fieldKey` appears in the incoming body. Return 400 if any are missing.

**`step.order` and `field.order`:** assign sequentially from the array index (0-based). Use the index position from the validated request array.

**Edge cases:**
- Template not found (GET/PATCH/DELETE): return 404
- PATCH body with duplicate `fieldKey` values within a step: Zod does not catch this — handle via `@@unique([stepId, order])` Prisma constraint; let Prisma throw and catch with a 400
- `prisma.$transaction` failure: propagate as 500

**`src/lib/form-validation.ts` consumers:** Task 3 (`src/app/api/templates/[id]/submit/route.ts`) imports `buildZodSchema`. Keep the function signature stable: `buildZodSchema(fields: Pick<FormField, 'fieldKey' | 'type' | 'required'>[])`.
