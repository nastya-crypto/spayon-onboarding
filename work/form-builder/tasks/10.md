# Task 10: Code Audit

## Required Skills

Before executing this task, load:
- `/skill:code-reviewing` — [skills/code-reviewing/SKILL.md](~/.claude/skills/code-reviewing/SKILL.md)

## Description

Full-feature code quality audit of the form-builder feature. Read all source files created or modified across Waves 1–4. Review holistically for cross-component issues that cannot be caught by single-file review: duplicate resource initialization, shared resource compliance with Architecture decisions, architectural consistency across all API routes and UI components, and TypeScript type safety.

This is a read-only audit task — no code is written or modified.

## What to do

1. Read all files listed in Context Files (feature source files).
2. Audit for each focus area:
   - **Duplicate resource initialization** — check that no route creates its own `PrismaClient`, Supabase client, or rate-limit `Map`; all must use singletons from `src/lib/`.
   - **Architectural consistency (admin routes)** — every admin API route (`/api/templates`, `/api/templates/[id]`, `/api/submissions`, `/api/submissions/[id]`, `/api/submissions/[id]/status`) must verify `session.user.role === 'ADMIN'` and return 403 on mismatch (Decision 9).
   - **Architectural consistency (public routes)** — `/api/templates/[id]/public` and `/api/templates/[id]/submit` must have no auth checks; submit must use rate limiter with key pattern `submit:${templateId}:${ip}` and IP from `x-forwarded-for` (Decision 5).
   - **Middleware allowlist** — `src/middleware.ts` must allowlist exact paths `/api/templates/:id/public` and `/api/templates/:id/submit` only; broader `/api/templates` prefix must NOT be allowlisted (Decision 7).
   - **Shared resource compliance** — Prisma via `src/lib/db.ts` singleton, Supabase via `src/lib/supabase.ts` singleton, rate-limit store via module-level `Map` in `src/lib/rate-limit.ts` (Architecture: Shared resources table).
   - **TypeScript type safety** — no untyped `any`, request/response shapes typed, Zod schemas present with `.max()` limits on all text inputs (Decision 10).
3. Write the audit report as JSON to `work/form-builder/logs/working/task-10/code-audit-report.json`.

## Acceptance Criteria

- [ ] All files listed under Context Files have been read
- [ ] No route initializes its own `PrismaClient`, Supabase client, or rate-limit `Map` — all use singletons from `src/lib/`
- [ ] Every admin API route verifies `session.user.role === 'ADMIN'` and returns 403 on mismatch
- [ ] Public routes (`/public`, `/submit`) have no auth checks
- [ ] Middleware allowlist uses exact paths only, not the broad `/api/templates` prefix
- [ ] All text inputs in admin routes have Zod `.max()` constraints
- [ ] No unresolved `any` types in route handlers or UI components
- [ ] Audit report written to `work/form-builder/logs/working/task-10/code-audit-report.json`
