# Task 11: Security Audit

## Required Skills

Before executing this task, load:
- `/skill:security-auditor` — [skills/security-auditor/SKILL.md](~/.claude/skills/security-auditor/SKILL.md)

## Description

Full-feature security audit of the form-builder feature. Reads all API routes and relevant UI files created or modified across Waves 1–4. Checks for OWASP Top 10 vulnerabilities with focus areas specific to this feature: broken access control on admin routes, injection via Zod validation limits and file upload handling, security misconfiguration in the middleware allowlist, rate limiting correctness, and XSS in the submission detail page.

This is a read-only audit task — no code is written or modified.

## What to do

1. Read all API routes:
   - `src/app/api/templates/route.ts`
   - `src/app/api/templates/[id]/route.ts`
   - `src/app/api/templates/[id]/public/route.ts`
   - `src/app/api/templates/[id]/submit/route.ts`
   - `src/app/api/submissions/route.ts`
   - `src/app/api/submissions/[id]/route.ts`
   - `src/app/api/submissions/[id]/status/route.ts`
   - `src/middleware.ts`

2. Read UI files:
   - `src/app/(onboarding)/onboarding/[templateId]/DynamicForm.tsx`
   - `src/app/(dashboard)/submissions/[id]/page.tsx`

3. Check **A01 — Broken Access Control**: every admin API route (`/api/templates`, `/api/templates/[id]`, `/api/submissions`, `/api/submissions/[id]`, `/api/submissions/[id]/status`) must verify `session.user.role === 'ADMIN'` and return 403 on mismatch (Decision 9).

4. Check **A03 — Injection / Input Validation**: all text inputs in admin routes use Zod `.max()` limits (Decision 10); file upload at `/api/templates/[id]/submit` enforces MIME allowlist (`image/jpeg`, `image/png`, `application/pdf` — SVG excluded) and filename sanitization pattern `name.replace(/[^a-zA-Z0-9._-]/g, '_')` (Decision 6).

5. Check **A05 — Security Misconfiguration**: `src/middleware.ts` allowlist uses exact paths for `/api/templates/:id/public` and `/api/templates/:id/submit` only; the broader `/api/templates` prefix must NOT be allowlisted (Decision 7).

6. Check **A07 — Rate Limiting**: rate limit at `/api/templates/[id]/submit` extracts IP via `req.headers.get('x-forwarded-for')?.split(',')[0]` (first value) to handle Railway reverse proxy correctly (Decision 5).

7. Check **XSS — Href validation**: in `src/app/(dashboard)/submissions/[id]/page.tsx`, FILE-type response hrefs must be validated to `http://` or `https://` only before rendering, to prevent `javascript:` URI injection (tech-spec Task 7 requirement).

8. Check **A09 — Logging / Error Handling**: error responses must not expose stack traces or internal implementation details.

9. Write the audit report as JSON to `work/form-builder/logs/working/task-11/security-audit-report.json`.

## Acceptance Criteria

- [ ] All admin routes verified to check `role === 'ADMIN'` and return 403 on mismatch
- [ ] MIME allowlist confirmed: `image/jpeg`, `image/png`, `application/pdf` only (no SVG)
- [ ] Filename sanitization confirmed at submit endpoint
- [ ] Middleware allowlist paths verified as exact (not broad `/api/templates` prefix)
- [ ] Rate limiting IP extraction verified: uses `x-forwarded-for` first value
- [ ] Href validation confirmed in submission detail page (`http`/`https` only)
- [ ] No stack traces or internal details in error responses
- [ ] Audit report written to `work/form-builder/logs/working/task-11/security-audit-report.json`

- `low` — defense-in-depth improvement, minor hardening opportunity
- `info` — observation, no action required
