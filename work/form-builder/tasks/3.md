# Task 3: Public form API

## Required Skills

Перед выполнением задачи загрузи:

## Description

Реализуем два публичных (без аутентификации) API-эндпоинта для работы с динамическими формами:

1. `GET /api/templates/[id]/public` — возвращает структуру шаблона (шаги + поля) для клиентского рендеринга формы. Поля `isProtected` и `fieldKey` **не включаются** в ответ — они внутренние и не должны утекать наружу.

2. `POST /api/templates/[id]/submit` — принимает заполненную форму (multipart/form-data), валидирует данные, загружает файлы в Supabase Storage (бакет `form-uploads`), создаёт `FormSubmission` и `FieldResponse` записи в БД. Требует rate limiting: 5 отправок с одного IP в час на шаблон, ключ `submit:${templateId}:${ip}`.

Оба эндпоинта публичные — клиент открывает форму без авторизации. Middleware обновляется точечно: добавляются только два конкретных пути, без широкого allowlisting-а `/api/templates`, чтобы не раскрыть админские CRUD-маршруты.

## What to do

1. Создать `src/app/api/templates/[id]/public/route.ts`:
   - `GET` handler — без аутентификации
   - Fetches template from DB with steps (ordered by `order`) and fields (ordered by `order`) using `prisma`
   - Returns 404 if template not found
   - Response shape: `{ id, name, steps: [{ id, title, order, fields: [{ id, label, type, placeholder, required, order }] }] }`
   - Fields `isProtected` and `fieldKey` must be explicitly excluded from the response

2. Создать `src/app/api/templates/[id]/submit/route.ts`:
   - `POST` handler — без аутентификации
   - Extract IP: `req.headers.get('x-forwarded-for')?.split(',')[0]?.trim() ?? 'unknown'` (Railway proxies via x-forwarded-for)
   - Rate limit: `rateLimit(`submit:${templateId}:${ip}`, 5, 60 * 60 * 1000)` — return 429 if not allowed
   - Fetch template with steps and fields; return 404 if not found
   - Parse `multipart/form-data` via `req.formData()`
   - Validate required fields using `buildZodSchema` from `src/lib/form-validation.ts` (created in T2)
   - For FILE-type fields: MIME allowlist `['image/jpeg', 'image/png', 'application/pdf']` (SVG excluded), max 10 MB, filename sanitized with `name.replace(/[^a-zA-Z0-9._-]/g, '_')` — same pattern as existing logo upload in `src/app/api/onboarding/submit/route.ts`
   - Upload files to Supabase Storage bucket `form-uploads`, path: `{submissionId}/{fieldId}-{timestamp}-{sanitizedFilename}`, store result as JSON `{ url, mimeType, size }` in `FieldResponse.value`
   - Extract `companyName` from the field with `isProtected: true` — return 400 if missing
   - Extract `email` from the first `EMAIL`-type field response if present — store as `null` if no EMAIL field
   - Create `FormSubmission` with snapshots: `templateName` = `template.name`, `companyName`, `email`
   - Create `FieldResponse` records for every submitted field: `fieldId`, `fieldLabel` = `field.label` (snapshot), `value`
   - Return 201 `{ success: true, submissionId }`

3. Обновить `src/middleware.ts`:
   - Add exact paths to the `authorized` callback allowlist:
     - `/api/templates/` paths matching `[id]/public` — use a regex or check pattern like `/^\/api\/templates\/[^/]+\/public$/`
     - `/api/templates/` paths matching `[id]/submit` — pattern `/^\/api\/templates\/[^/]+\/submit$/`
   - Do NOT add `/api/templates` broadly — that would bypass auth for all admin template management routes

## Details

**Files:**

- `src/app/api/templates/[id]/public/route.ts` (new) — GET handler: fetch template + steps + fields, strip `isProtected`/`fieldKey`, return 200/404
- `src/app/api/templates/[id]/submit/route.ts` (new) — POST handler: rate limit → fetch template → parse FormData → validate → upload files → create submission + responses → return 201
- `src/middleware.ts` — update `authorized` callback to allowlist exact public paths using regex matching against `pathname`

**Dependencies:**

- Task 1 (Prisma schema) must be complete — `FormTemplate`, `FormStep`, `FormField`, `FormSubmission`, `FieldResponse` models must exist
- Task 2 (Template management API) creates `src/lib/form-validation.ts` with `buildZodSchema` — if T2 is not yet complete, stub the validation inline and refactor once T2 merges

**Edge cases:**

- Template exists but has no steps → `GET /public` returns `{ steps: [] }` (valid response, not 404)
- Submit to template with no FILE fields → skip all file upload logic entirely
- File upload to Supabase fails (network error) → log error, return 500 (don't silently create partial submission with missing file)
- Multiple FILE fields in one submission → upload each independently; if any upload fails, return 500 (don't partially save)
- `x-forwarded-for` header absent → fall back to `'unknown'` as IP (same for all requests without header — they share a rate limit bucket, acceptable for MVP)
- `submissionId` needed for storage path before DB record exists → generate the submission ID before creating DB records (use `cuid()` or create submission first then upload)
- `companyName` field: find by `isProtected === true`, not by label string matching (labels can change)

**Implementation hints:**

- The `isProtected`/`fieldKey` exclusion in `/public` response: use explicit field selection in Prisma `select` clause rather than deleting keys after fetch — avoids accidentally leaking new fields added later
- Rate limit import: `import { rateLimit } from '@/lib/rate-limit'` — same as `src/app/api/onboarding/submit/route.ts`
- Supabase upload pattern is in `src/app/api/onboarding/submit/route.ts` lines 108–144 — reuse the same structure (arrayBuffer → upload → getPublicUrl)
- Middleware regex: `pathname.match(/^\/api\/templates\/[^/]+\/public$/)` and `pathname.match(/^\/api\/templates\/[^/]+\/submit$/)`
- `export const dynamic = "force-dynamic"` at top of both route files (matches existing pattern in the codebase)
- File size check (`file.size > MAX_FILE_SIZE`) must happen before `file.arrayBuffer()` to avoid loading oversized files into memory
