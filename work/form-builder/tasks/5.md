# Task 5: Templates admin pages

## Description

Build the template list page and template editor UI for the admin panel. The list page at `/dashboard/templates` shows all templates with Edit / Copy Link / Delete actions. The editor (shared `TemplateEditor` component) is used by both the New and Edit pages and allows admins to define template name, steps, and per-step fields.

This task depends on Task 2 (template management API) for data fetching/mutation and Task 4 (submissions API) for any submission-related counts. All UI text must be in English.

Key design constraints from tech-spec:
- **AC7:** Save is blocked if there are no steps OR any step has no fields ŌĆö validated in the UI before calling the API.
- **AC10:** Delete shows a confirm dialog "Are you sure?" before calling `DELETE /api/templates/[id]`.
- **Copy Link** uses `process.env.NEXTAUTH_URL` as the base URL (not `window.location.origin` ŌĆö the app runs behind a Railway reverse proxy).
- **isProtected fields:** The Delete button for a field is hidden when `field.isProtected === true` (Company Name field).

## What to do

1. Create `src/app/(dashboard)/templates/page.tsx` ŌĆö server component that:
   - Guards with `getServerSession` ŌåÆ redirect to `/login` if no session or not ADMIN/REVIEWER.
   - Fetches template list from `GET /api/templates` (returns `id, name, createdAt`).
   - Passes `NEXTAUTH_URL` as a server-side prop to the client table component so Copy Link can use it without `window.location.origin`.
   - Renders `TemplatesTable` (inline or extracted) with a "New Template" button linking to `/dashboard/templates/new`.

2. Create `src/app/(dashboard)/templates/new/page.tsx` ŌĆö server component that renders `TemplateEditor` in create mode (no initial data). On save, POSTs to `POST /api/templates` and redirects to `/dashboard/templates`.

3. Create `src/app/(dashboard)/templates/[id]/edit/page.tsx` ŌĆö server component that:
   - Fetches template by ID from `GET /api/templates/[id]`.
   - Renders `TemplateEditor` in edit mode with existing data.
   - On save, PATCHes `PATCH /api/templates/[id]` and redirects back to list.

4. Create `src/components/dashboard/TemplateEditor.tsx` ŌĆö `"use client"` component that:
   - Accepts `initialData?: { name: string, steps: Step[] }` and `onSave(data)` callback.
   - Has a template name input (required, max 255 chars).
   - Has a step list: each step shows title input + Ōåæ / Ōåō / Delete (step) buttons + field list.
   - "Add Step" button appends a new empty step.
   - Within each step: field list with Ōåæ / Ōåō / Delete (field) buttons (Delete hidden for `isProtected` fields).
   - "Add Field" button appends a new empty field to that step.
   - Field form: type dropdown (TEXT / EMAIL / URL / NUMBER / TEXTAREA / FILE / CHECKBOX), label input, placeholder input, required toggle.
   - Save button: disabled + shows "A template must have at least one step, and each step must have at least one field." inline message if validation fails (no steps, or any step has 0 fields).
   - On save, calls `onSave(data)` with the structured payload.

5. The templates list table must support:
   - Columns: Name, Created, Actions.
   - **Edit** ŌåÆ links to `/dashboard/templates/[id]/edit`.
   - **Copy Link** ŌåÆ copies `${baseUrl}/onboarding/${template.id}` to clipboard using `navigator.clipboard.writeText`. Show a brief "Copied!" feedback (toggle state, revert after 2s).
   - **Delete** ŌåÆ shows a native `window.confirm("Are you sure?")` dialog; on confirmation calls `DELETE /api/templates/[id]`, then refreshes the list (router.refresh() or local state removal).

## Details

**Files:**

- `src/app/(dashboard)/templates/new/page.tsx` (new) ŌĆö server component, renders `TemplateEditor` create mode

- `src/components/dashboard/TemplateEditor.tsx` (new) ŌĆö `"use client"` full editor component

**Dependencies:**
- Task 2: `GET /api/templates`, `POST /api/templates`, `GET /api/templates/[id]`, `PATCH /api/templates/[id]`, `DELETE /api/templates/[id]` must exist
- No new npm packages needed

**Edge cases:**
- Template with 0 steps fetched from API (legacy/corrupt data) ŌĆö editor handles gracefully, Save blocked
- `NEXTAUTH_URL` not set in env ŌĆö fall back to empty string; Copy Link result will be `/onboarding/[id]` (acceptable, not a runtime crash)
- `navigator.clipboard` unavailable (non-HTTPS localhost in some browsers) ŌĆö catch error and show fallback alert with URL text
- Step with title left empty ŌĆö API may reject; editor should validate step title is non-empty before save
- Ōåæ on first item / Ōåō on last item ŌĆö buttons should be disabled or no-op

**Implementation hints:**
- Use `useRouter` + `router.refresh()` (Next.js App Router) after successful delete to re-fetch server component data without full navigation
- Keep step/field state as arrays with client-generated temp IDs (e.g. `crypto.randomUUID()`) for React keys; strip them before sending to API
- Design tokens: primary blue-600, destructive red-500, card `bg-white rounded-xl shadow-sm`, inputs `rounded-lg border border-gray-300`, follow `MerchantsTable.tsx` Tailwind patterns
- `process.env.NEXTAUTH_URL` is only available server-side; pass it as a prop to the client `TemplatesTable` component from the server page
