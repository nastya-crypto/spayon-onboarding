## Required Skills

Before executing this task, load:

## Description

Switch the admin dashboard from showing `Merchant` records to showing `FormSubmission` records. The dashboard currently queries `prisma.merchant` and renders `MerchantsTable` + `CreateLinkButton`. After this task it must query `GET /api/submissions` (created in Task 4) and render a new `SubmissionsTable` component alongside `StatsCard` counts driven by `SubmissionStatus` values.

`CreateLinkButton` is replaced by a simple `<Link>` to `/dashboard/templates` with the text "Templates". `StatsCard` labels switch from Russian to English and counts switch from `Merchant.status` to `FormSubmission.status` (NEW / IN_REVIEW / APPROVED / REJECTED). All UI-visible text must be in English.

This task depends on Task 2 (template management API) and Task 4 (submissions API) being complete. The new `SubmissionsTable` component lives in `src/components/dashboard/MerchantsTable.tsx` ‚Äî the file is modified in-place (renamed component export, not a new file) to keep the import surface minimal.

## What to do

1. **`src/app/(dashboard)/dashboard/page.tsx`**
   - Replace `getMerchantsData()` with `getSubmissionsData()` that calls `fetch('/api/submissions', { cache: 'no-store' })` (or direct Prisma query matching the shape returned by Task 4's `GET /api/submissions`).
   - The data shape needed: `{ id, companyName, templateName, createdAt, status }[]` plus status counts `{ NEW, IN_REVIEW, APPROVED, REJECTED }`.
   - Replace `<MerchantsTable merchants={...} />` with `<SubmissionsTable submissions={...} />`.
   - Replace `<CreateLinkButton />` with `<Link href="/dashboard/templates" className="...">Templates</Link>` (use `next/link`).
   - Update `StatsCard` titles to English: "New", "In Review", "Approved", "Rejected".
   - Update page heading and subheading to English: "Submissions" / "Manage client onboarding submissions".
   - Remove unused imports (`CreateLinkButton`, `MerchantsTable`, `prisma.merchant` calls).

2. **`src/components/dashboard/MerchantsTable.tsx`**
   - Rename the component from `MerchantsTable` to `SubmissionsTable` (update export name and all internal references).
   - Replace `MerchantRow` type with `SubmissionRow`: `{ id: string; companyName: string; templateName: string; status: "NEW" | "IN_REVIEW" | "APPROVED" | "REJECTED"; createdAt: string }`.
   - Remove the `user` / `email` column ‚Äî it is excluded from the list response (tech-spec Decision ‚Äî email excluded from list).
   - Add a `templateName` column after `companyName`.
   - Update table headers to English: "Company", "Template", "Status", "Date", "Actions".
   - Update the search filter to match on `companyName` and `templateName` (replacing the old `businessName` + `user.email` filter).
   - Update the "Open" link href from `/merchants/${id}` to `/dashboard/submissions/${id}`.
   - Update all UI text to English: search placeholder "Search by company or template...", empty state "No submissions yet" / "Submit a form to see it here".
   - Update `STATUS_LABELS` to English: NEW ‚Üí "New", IN_REVIEW ‚Üí "In Review", APPROVED ‚Üí "Approved", REJECTED ‚Üí "Rejected".
   - Keep the existing status badge colour mapping (`STATUS_STYLES`) unchanged.

3. **`src/components/dashboard/CreateLinkButton.tsx`**
   - Replace the entire component body with a simple re-export or inline `<Link>` wrapper so the file is no longer a dead-code island. Since `dashboard/page.tsx` will import `Link` directly, this file can be simplified to a thin named export `TemplatesButton` that renders `<Link href="/dashboard/templates">Templates</Link>` ‚Äî or left as a stub. The important thing is that it no longer calls `/api/onboarding/create-link` (that endpoint is deleted in Task 9). Keep the file present so Task 9 can delete it cleanly.

4. **`src/components/dashboard/StatsCard.tsx`**
   - Change the badge label from Russian "–∑–∞—è–≤–æ–∫" to English "submissions".
   - The `StatsCardProps` interface is unchanged ‚Äî `title` is passed from the parent, so no prop changes needed.
